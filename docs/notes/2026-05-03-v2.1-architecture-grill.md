- **인프라**: docker-compose 로컬. EKS·AWS·Helm·Terraform·External Secrets 폐기

## 6. 문서별 변경 사항

### 6.1 `docs/decisions/001-olap-and-infra-roadmap.md` 재작성

- §Context: "운영 중 MVP" → "로컬 프로토타입, 평가용 졸업과제"
- §Decision 2: LISTEN/NOTIFY → **Outbox pattern + Kafka KRaft 단일 노드 (fallback Redpanda)**. 운영 트랜잭션 안에서 outbox INSERT, publisher 프로세스가 LISTEN-poll → Kafka 토픽 발행
- §Decision 3: AWS EKS 이관 → **로컬 docker-compose 환경. 확장성은 설계 문서·다이어그램·worker scale=N 라이브 시연으로 입증**
- §Decision 6: RAG 챗봇 마이크로서비스 → **단일 엔드포인트 (`/api/v1/chat`). 분석 SQL 결과 + LLM 호출. 단순 이름 치환 마스킹. 데모용**
- §Roadmap: Phase 5 (EKS) 제거. Phase 4 (컨테이너화)는 docker-compose로 통합
- §References: 평가 rubric 인용 (메시지 스트림 가점, EKS 비필수) 추가

### 6.2 `docs/decisions/002-cdc-pattern-outbox-kafka.md` 신규

- **Context**: 메시지 스트림 가점 명시, 1인 + 미경험 + 2개월 자원 제약
- **Decision**: Outbox table + Kafka KRaft + app publisher (aiokafka)
- **Alternatives Considered**: A(Debezium), B(Outbox), C(App-direct) 표
- **Consequences**: 운영 트랜잭션에 outbox INSERT 추가 (latency 영향 측정 필요), publisher 프로세스 신설, Kafka 학습 부담
- **Fallback**: Week 3 금요일 PoC fail 시 Redpanda 전환 (Kafka API 호환, publisher/consumer 코드 변경 0, docker-compose만 교체)

### 6.3 `docs/prd.md` v2.1 (재작성)

- §1·§7 Stack 표: EKS · Helm · ALB · Terraform · External Secrets 라인 삭제 → "**docker-compose (로컬 평가)**"로 통일
- §9 RISK 테이블:
  - RISK-004 (Render cold start): 폐기 또는 "로컬 부팅 시간"으로 변경
  - RISK-005: "publisher 다운 시 outbox 미발행 → 부팅 시 unsent row catch-up"으로 재서술
  - RISK-006 (가점 미충족): 폐기
  - RISK-008 (EKS 운영 부담): 폐기
  - **RISK-009 신규**: "Kafka 미경험 학습 부담 (W3 PoC fail 시 Redpanda fallback)"
- §11 챗봇:
  - "RAG" 표현 제거 → "AI 어시스턴트 (데모)"
  - 마이크로서비스·provider 추상화 표기 제거
  - PII 정책: 챗봇 답변 범위를 학급 단위 통계로 제한 (k-익명성 임계 실질 충족)
  - REQ-080·081·082 우선순위 재검토 (Should → Could 검토)

### 6.4 `docs/design-spec.md` v2.1 (재작성)

- §1 다이어그램: EKS Cluster 박스 → "**docker-compose** (FastAPI · Outbox publisher · analytics-worker · Kafka KRaft · 단일 챗봇 엔드포인트 통합)"
- §9 Analytics Layer:
  - 트리거 SQL (`pg_notify`) → **Outbox 테이블 DDL + 운영 라우터의 outbox INSERT 코드 + publisher pseudo-code (aiokafka)**
  - Worker pseudo-code: LISTEN → **Kafka consumer (aiokafka), consumer group, offset commit**
- §10 Chatbot:
  - 마이크로서비스 표기 제거
  - Sanitizer를 단순 함수로 다운사이즈 (이름 → "학생A")
  - LLMClient 추상화 제거 (단일 provider 직접 호출)

### 6.5 `docs/architecture.md` v1.1 (재작성)

- §1: 기반 문서 "ADR-001 재작성 + ADR-002 신규" 명시
- §3 컨테이너 다이어그램: "AWS EKS Cluster" 박스 → "docker-compose"
- §4.1 데이터 흐름 (운영): `pg_notify(analytics_events)` → "**outbox INSERT (같은 트랜잭션) → publisher LISTEN/poll → Kafka producer.send('grade_events') → consumer (analytics-worker) → analytics.* UPSERT**"
- §5 확장성:
  - HPA 표현 제거
  - "**consumer group + 파티션 수평 확장. docker-compose `--scale analytics-worker=3` 데모로 입증**"
- §6 B2: NOTIFY payload 손실 → "**publisher 다운 시 outbox row unsent → 부팅 시 `WHERE sent_at IS NULL` catch-up**"
- §11 OQ-101: 폐기 (평가 전 Kafka 도입 확정)
